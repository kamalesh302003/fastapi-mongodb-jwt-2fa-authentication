# 🔐 FastAPI + MongoDB Two-Factor Authentication System

A secure authentication system built using **FastAPI**, **MongoDB**, **JWT**, and **Two-Factor Authentication (2FA)**. This project provides a complete user authentication workflow with secure password hashing, OTP verification, QR code generation, Google Authenticator integration, and protected dashboard access.

---

## 🚀 Features

* User Registration and Login
* Secure Password Hashing
* JWT-Based Authentication
* Cookie-Based Session Management
* Two-Factor Authentication (2FA)
* Time-Based One-Time Passwords (TOTP)
* QR Code Generation for Google Authenticator
* Enable / Disable 2FA
* Protected Dashboard Routes
* MongoDB Integration
* Jinja2 Template Rendering
* FastAPI RESTful Architecture

## ⚙️ Installation

### 1. Clone Repository

```bash
git clone https://github.com/yourusername/fastapi-mongodb-jwt-2fa-authentication.git

cd fastapi-mongodb-jwt-2fa-authentication
```

### 2. Create Virtual Environment

```bash
python -m venv venv
```

### Activate Virtual Environment

Windows:

```bash
venv\Scripts\activate
```

Linux / macOS:

```bash
source venv/bin/activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

## 🗄️ Configure MongoDB

Make sure MongoDB is running locally.

Example database configuration:

```python
from motor.motor_asyncio import AsyncIOMotorClient

client = AsyncIOMotorClient("mongodb://localhost:27017")

db = client["two_factor_auth"]

users_collection = db["users"]
```

## 🔑 Authentication Workflow

### User Registration

1. Create account
2. Password is hashed before storage
3. User data stored in MongoDB

### First Login

1. User enters username and password
2. System redirects to 2FA setup page
3. QR code generated
4. Scan QR using Google Authenticator
5. Verify OTP
6. Enable Two-Factor Authentication

### Subsequent Login

1. Enter username and password
2. Enter OTP from Google Authenticator
3. JWT token generated
4. Access protected dashboard

---

## 🔐 Security Features

### Password Hashing

Passwords are securely hashed before storing in MongoDB.

### JWT Authentication

Secure access tokens are generated for authenticated users.

### Two-Factor Authentication

Uses Time-Based One-Time Passwords (TOTP) generated by:

* Google Authenticator
* Microsoft Authenticator
* Authy

### Protected Routes

Dashboard access is restricted to authenticated users only.

1. Login with username and password.
2. Scan QR code using Google Authenticator.
3. Enter generated OTP.
4. Enable 2FA.
5. Use OTP for future logins.

## 📡 API Endpoints

| ------ | --------------------- | --------------------- |
